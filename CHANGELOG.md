## [v0.2.0] - 2026-07-16

### Highlights

- New `render` package: provider-agnostic batch avatar video generation (submit narration audio, poll, download talking-head MP4)
- Package `avatar` renamed to `live` so the real-time and batch surfaces are self-describing

### Added

- `render.Provider` interface with `Generate`, `Status`, and `Download` for asynchronous video generation ([`7585f63`](https://github.com/plexusone/omniavatar-core/commit/7585f63))
- `render.AudioUploader` optional capability interface for providers that can host local audio files ([`7585f63`](https://github.com/plexusone/omniavatar-core/commit/7585f63))
- `render.GenerateRequest` with `AudioURL` (primary, drives lip-sync from existing narration) and `Script` (secondary, provider TTS) inputs plus provider-specific `Extensions` ([`7585f63`](https://github.com/plexusone/omniavatar-core/commit/7585f63))
- `render.Job`, `render.JobState` (with `Terminal()`), and `render.JobStatus` with normalized state plus provider-native `RawStatus` ([`7585f63`](https://github.com/plexusone/omniavatar-core/commit/7585f63))
- `render.Wait` polling helper with context cancellation and configurable interval ([`7585f63`](https://github.com/plexusone/omniavatar-core/commit/7585f63))
- Render sentinel errors: `ErrAudioUploadUnsupported`, `ErrInvalidRequest`, `ErrJobNotFound`, `ErrJobFailed`, `ErrJobNotCompleted`, `ErrProviderUnavailable`, `ErrProviderAuthFailed`, `ErrInvalidConfig` ([`7585f63`](https://github.com/plexusone/omniavatar-core/commit/7585f63))
- `registry.RenderProviderFactory` type for render provider registration ([`4f4c725`](https://github.com/plexusone/omniavatar-core/commit/4f4c725))
- Unit tests for the `render` package (`Wait`, state transitions, request validation) ([`b20c0b6`](https://github.com/plexusone/omniavatar-core/commit/b20c0b6))

### Changed

- **BREAKING:** package `avatar` renamed to `live`; update imports to `github.com/plexusone/omniavatar-core/live` (type and function names unchanged) ([`4f4c725`](https://github.com/plexusone/omniavatar-core/commit/4f4c725))
- **BREAKING:** `registry.ProviderFactory` renamed to `registry.LiveProviderFactory` ([`4f4c725`](https://github.com/plexusone/omniavatar-core/commit/4f4c725))
- **BREAKING:** error string prefixes changed from `avatar:` to `live:` and `ProviderError` prefix from `avatar/` to `live/` ([`4f4c725`](https://github.com/plexusone/omniavatar-core/commit/4f4c725))

### Documentation

- README.md and doc.go updated to document both `live` and `render` surfaces ([`3e00d7c`](https://github.com/plexusone/omniavatar-core/commit/3e00d7c))
- MkDocs site added (interfaces reference for both surfaces, architecture, releases) with a `Docs` GitHub Actions workflow deploying via `mkdocs gh-deploy` ([`0d753ea`](https://github.com/plexusone/omniavatar-core/commit/0d753ea))

## [v0.1.0] - 2026-07-12 ([`42e9e9a`](https://github.com/plexusone/omniavatar-core/commit/42e9e9a))

### Highlights

- Initial release of omniavatar-core interfaces
- Provider-agnostic avatar abstraction for real-time AI avatars
- Support for HeyGen, Tavus, and bitHuman avatar providers
